# 学期末总 Checklist（按优先级排序）

## 总体目标
本学期末的核心策略是：**优先保证高价值、高截止压力、高认知负荷的任务顺利完成；持续推进责任型项目；同时为中长期研究布局留出不断线的空间。**

---

# 一、优先级总排序

## Tier 1：最高优先级（必须打赢）
1. **Telework 论文（thesis + 投稿准备）**
2. **Remote sensing 识别项目**
3. **课程作业收尾**

## Tier 2：中高优先级（必须稳住）
4. **Grant fund tracking**
5. **Erick RA：SEPTA project personnel management**
6. **Anchor Institution project（Eugenie Birch）**

## Tier 3：中长期布局（必须启动，但不必做深）
7. **Arena research：proposal + 潜在合作者沟通**

---

# 三、各项目完整 Checklist

---

## 1. Telework 论文（最高优先级）

### 总目标
- [ ] 完成 literature review 修订
- [ ] 写完 results section
- [ ] 写完 discussion section
- [ ] 完成 thesis 当前版本整合
- [ ] 准备 6 月初投稿 TRA 或 TRB

### A. Literature Review
- [ ] 修订文献综述整体结构
- [ ] 明确 telework / WFH / remote work 的概念界定
- [ ] 检查 literature review 中是否存在薄弱 citation
- [ ] 强化文章与现有文献的差异和贡献
- [ ] 补足关于非线性、行为机制、日内出行重组的 framing
- [ ] 统一概念、术语和写作语气
- [ ] 做最后一轮 citation validity check

### B. Results
- [ ] 完成主模型结果写作
- [ ] 完成 supplementary model 结果写作
- [ ] 完成 mechanism models 结果写作
- [ ] 将 PSM / robustness test 结果写入适当位置
- [ ] 检查表格、图表与正文解释是否一致
- [ ] 明确正文与 appendix 的分工

### C. Discussion
- [ ] 解释 telework intensity 的核心发现
- [ ] 讨论 total trips 与不同 trip purpose 的关系
- [ ] 讨论 work-related trips 与 non-work trips 的机制差异
- [ ] 讨论研究的理论意义
- [ ] 讨论政策含义
- [ ] 讨论研究局限性
- [ ] 指向未来研究方向

### D. 投稿准备
- [ ] 对照 TRA / TRB 风格调整文章结构
- [ ] 修改 abstract
- [ ] 修改标题备选版本
- [ ] 检查参考文献格式
- [ ] 检查表格、附录、图注格式
- [ ] 整理 cover letter 草稿
- [ ] 确定优先投稿目标（TRA 或 TRB）
- [ ] 预留一轮全文通读和润色时间

### 本项目提醒
- [ ] 避免在单个段落上过度打磨，优先完成整篇闭环
- [ ] 优先保证 “文献综述—结果—讨论—投稿准备” 的主线推进
- [ ] 这是当前最重要的学术产出，优先占据最好的专注时间

---

## 2. Remote Sensing 识别项目

### 总目标
- [ ] **2 周内快速推进**
- [ ] **3 周内完成**
- [ ] 明确 workflow 并产出可执行结果
- [ ] 聚焦于 **cross width** 和 **crosswalk quality（painting）**
- [ ] 完成方法、结果、文档整理

### A. 研究目标与方案锁定
- [ ] 最终确认研究对象：cross width / crosswalk quality
- [ ] 明确不再扩展到 road width 等额外任务
- [ ] 明确分析流程（preprocessing → sample prep → segmentation/identification → evaluation）
- [ ] 建立清晰的步骤清单和方法说明

### B. 数据准备
- [ ] 整理 imagery 数据
- [ ] 整理辅助道路 / crossing 数据
- [ ] 选择局部 test area
- [ ] 做 preprocessing
- [ ] 建立样本管理文件夹结构
- [ ] 准备训练 / 测试数据

### C. 模型 / 方法推进
- [ ] 跑通 baseline workflow
- [ ] 完成第一次识别结果
- [ ] 检查 cross width 提取效果
- [ ] 检查 crosswalk painting 识别效果
- [ ] 识别主要误差来源
- [ ] 调整 segmentation / identification 方法
- [ ] 完成优化后版本

### D. 结果评估与导出
- [ ] 完成初步精度检查
- [ ] 整理典型成功案例
- [ ] 整理典型失败案例
- [ ] 明确哪些问题可接受，哪些必须修正
- [ ] 输出街道层级结果
- [ ] 保存结果图、参数、日志

### E. 文档整理
- [ ] 整理英文 checklist 和 methodology
- [ ] 保存代码版本
- [ ] 保存参数设置
- [ ] 保存数据处理流程记录
- [ ] 形成一个可复现版本

### 三周推进节奏
#### 第 1 周
- [ ] 锁定方案
- [ ] 建立 test area
- [ ] 整理数据
- [ ] 跑通 baseline

#### 第 2 周
- [ ] 完成第一次结果
- [ ] 识别主要问题
- [ ] 做方法调整
- [ ] 输出阶段性图例和案例

#### 第 3 周
- [ ] 完成最终版本
- [ ] 整理结果
- [ ] 完成方法文档
- [ ] 完成交付材料整理

### 本项目提醒
- [ ] 避免无限优化
- [ ] 目标是完成一个可信、可复现、能解释的版本
- [ ] 优先完成闭环，再考虑提高精度
